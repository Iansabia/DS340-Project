<!-- Ticker format analysis from active_matches.json (615 pairs): -->

Format 1 - Daily with YYMMDD date + threshold:
  KXWTI-26APR08-T105.99       -> resolves April 8, 2026
  KXWTI-26APR07-T111.99       -> resolves April 7, 2026

Format 2 - Monthly with YYMMDD embedded:
  KXBTCMAXMON-BTC-26APR30-7500000  -> resolves April 30, 2026
  KXBTCMINMON-BTC-26APR30-6500000  -> resolves April 30, 2026

Format 3 - Yearly with YYMMMDD:
  KXBTCMAXY-27JAN01-50000.00  -> resolves January 1, 2027
  KXBTCMINY-27JAN01-50000.00  -> resolves January 1, 2027

Format 4 - Monthly Fed meetings with YYMM:
  KXFEDDECISION-26APR-C25     -> resolves end of April 2026
  KXFEDDECISION-27JAN-H0      -> resolves end of January 2027
  KXFEDCOMBO-26APR26JUN-C2    -> resolves June 2026 (later date)

Format 5 - Year-only (politics, 2028 elections):
  KXPRESNOMD-28-AOC           -> resolves ~Nov 2028 (election day)
  KXPRESNOMR-28-ETRU          -> resolves ~Aug 2028 (convention)
  KXVPRESNOMD-28-RW           -> resolves ~Aug 2028 (convention)

Format 6 - Sports season (year only):
  KXNBA-26-CHA                -> resolves ~June 2026 (NBA finals)
  KXMLB-26-MIA                -> resolves ~Oct 2026 (World Series)
  KXF1-26-RB                  -> resolves ~Dec 2026 (season end)
  KXMENWORLDCUP-26-US         -> resolves ~July 2026 (tournament end)
  KXUCL-26-BAY                -> resolves ~June 2026 (UCL final)
  KXLALIGA-26-BAR             -> resolves ~May 2026 (season end)

Format 7 - Specific event dates with YYMMMDD:
  KXTRUMPCHINA-26-MAY15       -> resolves May 15, 2026
  KXBRAZILPRES1R-26OCT04      -> resolves October 4, 2026
  KXHUNGARYPARLI-26APR07      -> resolves April 7, 2026

Format 8 - Special/unclear:
  KXFEDCHAIRCONFIRM-JSHE      -> no date, use Kalshi API close_date
  KXPERSONPRESMAM-45           -> no date parseable
  KXAOCSENATE-28              -> year only

Distribution: 161 YYMMDD, 65 YYMM, 354 year-only, 35 unclear

<!-- Tier classification thresholds from CONTEXT.md: -->
MONITORING:  > 168 hours (7+ days)    -> 4h bars
WARMING:     72-168 hours (3-7 days)  -> 1h bars
ACTIVE_WEEK: 24-72 hours (1-3 days)  -> 15min bars
ACTIVE_DAY:  6-24 hours              -> 5min bars
SPRINT:      2-6 hours               -> 5min bars + exit flag
NO_TRADE:    < 2 hours               -> no new entries

    - Define Tier enum: MONITORING, WARMING, ACTIVE_WEEK, ACTIVE_DAY, SPRINT, NO_TRADE, UNKNOWN
    - Define BAR_INTERVALS dict: {Tier.MONITORING: 14400, Tier.WARMING: 3600, Tier.ACTIVE_WEEK: 900, Tier.ACTIVE_DAY: 300, Tier.SPRINT: 300, Tier.NO_TRADE: None, Tier.UNKNOWN: 14400}
    - parse_resolution_date(ticker: str) -> datetime | None:
      - Use regex to extract date components from ticker string
      - Pattern priority: try YYMMDD first (r'(\d{2})([A-Z]{3})(\d{2})'), then YYMM (r'-(\d{2})([A-Z]{3})-'), then year-only (r'-(\d{2})-')

      - Return None if no pattern matches
    - classify_contract(resolution_date: datetime | None, now: datetime) -> Tier:
      - If resolution_date is None: return Tier.UNKNOWN
      - Compute hours_remaining = (resolution_date - now).total_seconds() / 3600
      - Apply thresholds from CONTEXT.md: <2h NO_TRADE, <6h SPRINT, <24h ACTIVE_DAY, <72h ACTIVE_WEEK, <168h WARMING, else MONITORING
    - classify_all_pairs(matches: list[dict], now: datetime | None = None) -> dict[str, dict]:
      - For each match in matches, parse resolution_date from kalshi_ticker
      - Classify into tier
      - Return dict: {pair_id: {"tier": tier.value, "resolution_date": iso_str_or_none, "hours_remaining": float, "bar_interval_seconds": int_or_none}}
