    Create `src/live/trading_cycle.py` -- the main entry point for the Oracle VM cron job.

    **Key design: tier-aware collection cadence.**
    Each cron run (every 5 min) checks which pairs need a new bar:

    ```python
    def should_collect(pair_id: str, tier: str, last_collection_times: dict, now: datetime) -> bool:
        """Check if enough time has elapsed since last collection for this pair's tier."""
        bar_interval = BAR_INTERVALS.get(Tier(tier))
        if bar_interval is None:
            return False  # NO_TRADE tier
        last_time = last_collection_times.get(pair_id)
        if last_time is None:
            return True  # never collected
        elapsed = (now - last_time).total_seconds()
        return elapsed >= bar_interval
    ```

    **Main function flow:**
    1. Parse args: --live-dir, --model-dir, --dry-run
    2. Guard against PyTorch imports: at top of file, check that torch is NOT imported. If accidentally imported via a transitive dependency, log error and exit. Add explicit check:
       ```python
       import sys
       if 'torch' in sys.modules:
           logger.error("PyTorch loaded! This will exceed 1GB RAM. Fix imports.")
           sys.exit(1)
       ```
    3. Load last_collection_times from data/live/collection_times.json
    4. Run ContractClassifier on active_matches.json
    5. Filter to pairs that should_collect (based on tier + elapsed time)
    6. For the filtered pairs only, fetch prices via direct API calls (NOT LiveCollector which imports all of src.live -- use inline requests to avoid transitive torch imports)
    7. Build bars, update positions, check exits, check entries via TradingStrategy
    8. Save updated collection_times.json
    9. Log: "Cycle complete: {N} pairs collected, {M} entries, {K} exits, {J} open positions"

    **Lightweight price fetching (avoid LiveCollector's heavy imports):**
    Rather than importing LiveCollector (which may pull in heavy deps), copy the price fetching logic into trading_cycle.py or import only the specific fetch functions. The key is: no transitive PyTorch imports.

    Actually, since LiveCollector does NOT import torch (it only uses requests/pandas/numpy), it's safe to import. But verify that `from src.live.collector import LiveCollector` doesn't trigger any torch imports. If it does (because of __init__.py re-exports), use direct imports.

    **collection_times.json format:**
    ```json
    {

      "live_0042": "2026-04-08T14:25:00Z"
    ```

    **scripts/setup_oracle.sh:**
    One-command deployment script for Oracle Linux 9 VM. User runs:
    `ssh opc@129.80.4.112 'bash -s' < scripts/setup_oracle.sh`

    Script contents:
    ```bash
    #!/bin/bash
    set -euo pipefail

    echo "=== DS340 Oracle VM Setup ==="

    # 1. Install Python 3.12 + git
    sudo dnf install -y python3.12 python3.12-pip git

    # 2. Clone repo (or pull if exists)
    REPO_DIR="$HOME/DS340-Project"
    if [ -d "$REPO_DIR" ]; then
        cd "$REPO_DIR" && git pull --rebase
    else
        git clone https://github.com/iansabia/DS340-Project.git "$REPO_DIR"
        cd "$REPO_DIR"
    fi

    # 3. Create venv with ONLY lightweight deps (no torch!)
    python3.12 -m venv .venv-oracle
    source .venv-oracle/bin/activate

    # 4. Install minimal requirements
    pip install --upgrade pip
    pip install pandas numpy scikit-learn xgboost requests

    # 5. Verify no torch
    python -c "import sys; assert 'torch' not in sys.modules; print('No torch: OK')"

    # 6. Verify models load
    python -c "
    from src.models.base import BasePredictor
    lr = BasePredictor.load('models/deployed/linear_regression.pkl')
    print(f'Model loaded: {lr.name}')
    "

    # 7. Set up cron (every 5 minutes)
    CRON_CMD="*/5 * * * * cd $REPO_DIR && .venv-oracle/bin/python -m src.live.trading_cycle >> logs/trading_cycle.log 2>&1"
    COMMIT_CMD="0 * * * * cd $REPO_DIR && git add -A data/live/ && git commit -m 'auto: hourly data commit' --allow-empty && git push origin master 2>> logs/git_push.log"

    # Create logs directory
    mkdir -p "$REPO_DIR/logs"

    # Write crontab (preserve existing entries)
    (crontab -l 2>/dev/null | grep -v "trading_cycle\|hourly data commit"; echo "$CRON_CMD"; echo "$COMMIT_CMD") | crontab -

    echo "=== Cron installed ==="
    crontab -l

    # 8. Test one cycle (dry run)
    python -m src.live.trading_cycle --dry-run

    echo "=== Setup complete ==="
    echo "VM will run trading cycles every 5 minutes"
    echo "Auto-commit every hour"
    echo "Logs: $REPO_DIR/logs/trading_cycle.log"
    ```

    **Important notes for the script:**
    - Use python3.12 (not python3 which may be 3.6 on Oracle Linux 9)
    - The .venv-oracle is separate from the development .venv to avoid torch contamination
    - git clone URL should match the actual repo URL (user may need to update)
    - The script is idempotent: safe to re-run
