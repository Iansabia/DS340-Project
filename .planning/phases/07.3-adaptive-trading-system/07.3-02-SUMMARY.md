## Accomplishments
- Position dataclass and PositionManager with full SQLite persistence (WAL mode, positions + closed_positions tables)
- 5 exit rules with concrete thresholds: take-profit (50% convergence), stop-loss (130% of entry), momentum (3 consecutive adverse bars), time-stop (4/24/168/500 bars by tier), resolution proximity (<24h)
- Direction-aware P&L for both short_spread and long_spread positions
- 32 passing TDD tests covering lifecycle, P&L, persistence, all exit rules individually, boundary values, and priority ordering

## Decisions Made
- Exit rules check priority: RESOLUTION > STOP_LOSS > TAKE_PROFIT > MOMENTUM > TIME_STOP (safety exits first)
- Spread history stored as JSON array in SQLite TEXT column, kept to last 10 entries
- SQLite WAL mode enabled for concurrent read safety during cron-based position updates
- JSONL append-only backup provides crash recovery independent of SQLite
