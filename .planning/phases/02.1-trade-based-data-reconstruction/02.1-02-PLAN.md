**Create src/data/aligner.py** with two public functions:

`align_pair(kalshi_candles: pd.DataFrame, poly_candles: pd.DataFrame, pair_id: str) -> pd.DataFrame | None`:
- Both inputs are DataFrames with CANDLE_COLUMNS (from reconstruct_candles)
- If either is empty, return None
- Determine time range: `min_ts = min(kalshi_candles["timestamp"].min(), poly_candles["timestamp"].min())`, `max_ts = max(kalshi_candles["timestamp"].max(), poly_candles["timestamp"].max())`
- Create unified 4-hour grid: `grid_ts = np.arange(min_ts, max_ts + 14400, 14400)` (ensure bar-aligned by flooring: `min_ts = (min_ts // 14400) * 14400`)
- Create grid DataFrame: `grid = pd.DataFrame({"timestamp": grid_ts})`
- For each platform (kalshi, poly):
  - Merge candles onto grid with left join on timestamp
  - Prefix all CANDLE_COLUMNS (except timestamp) with platform name: e.g., `kalshi_vwap`, `kalshi_open`, ...
  - Forward-fill with limit=6 (24 hours / 4-hour bars)
  - Compute `{platform}_hours_since_last_trade`: for each row, count bars since last `has_trade==True` row, multiply by 4. If current bar has_trade, value is 0. If never had a trade, value is NaN.
  - After forward-fill, set `{platform}_has_trade` to False for filled bars (distinguish fresh vs filled)
- Merge both platform grids on timestamp
- Compute `spread = kalshi_vwap - polymarket_vwap`
- Add `pair_id` column
- Return the merged DataFrame

`align_all_pairs(pairs: list[dict], candles_dir: Path, bar_seconds: int = 14400) -> tuple[pd.DataFrame, dict]`:
- For each pair in pairs:
  - Load candles from `{candles_dir}/kalshi/{kalshi_market_id}_candles.parquet` and `{candles_dir}/polymarket/{polymarket_market_id}_candles.parquet`
  - If either file missing or empty -> add to report with reason "missing_candles", skip
  - Count total trades: sum of trade_count column. If either platform < 20 -> add to report with reason "insufficient_trades", skip
  - Call align_pair(kalshi_candles, poly_candles, pair_id)
  - If result is None -> add to report with reason "alignment_failed", skip
  - Compute staleness_ratio = fraction of rows where BOTH kalshi_has_trade and polymarket_has_trade are False
  - If staleness_ratio >= 0.80 -> add to report with reason "too_stale", skip
  - Compute mean_abs_spread = mean(|spread|) where spread is not NaN
  - If mean_abs_spread >= 0.30 -> add to report with reason "spread_not_mean_reverting", skip
  - Add to aligned list
- Concatenate all aligned DataFrames
- Build quality_report dict:
  ```python
  {
      "total_pairs": len(pairs),
      "aligned_pairs": N,
      "excluded_pairs": N,
      "exclusion_reasons": {"missing_candles": N, "insufficient_trades": N, "too_stale": N, "spread_not_mean_reverting": N, "alignment_failed": N},
      "per_pair": [{"pair_id": ..., "status": "included"|"excluded", "reason": ..., "kalshi_trades": N, "polymarket_trades": N, "staleness_ratio": float, "mean_abs_spread": float, "bars": N}, ...]
  }
  ```
- Return (concatenated_df, quality_report)

**Create scripts/rebuild_data.py** -- CLI entry point that runs the full trade-based data reconstruction pipeline.

The script has 4 stages, each logged clearly:

```python
#!/usr/bin/env python3
"""Rebuild data pipeline: fetch trades -> reconstruct candles -> align pairs -> save.

Usage:
    python scripts/rebuild_data.py [--pairs-file PATH] [--output-dir PATH] [--skip-fetch]
"""
```

**Stage 1: Load pairs**
- Load pairs from `data/processed/accepted_pairs.json` (or --pairs-file override)
- Log: "Loaded {N} accepted pairs"

**Stage 2: Fetch raw trades** (skippable with --skip-fetch)
- Create ResilientClient instances:
  - Kalshi: base_url=`https://api.elections.kalshi.com/trade-api/v2`, requests_per_second=18.0
  - Polymarket: base_url=`https://data-api.polymarket.com`, requests_per_second=2.0 (conservative for Data API)
- Call `fetch_and_save_trades(pairs, kalshi_client, poly_client, output_dir / "raw")`
- Log stats from return dict

**Stage 3: Reconstruct candles**
- For each unique market across all pairs (both platforms):
  - Load trades from `{output_dir}/raw/{platform}/{market_id}_trades.parquet`
  - If file missing or empty, skip (will be caught by aligner quality check)
  - Call `reconstruct_candles(trades_df, bar_seconds=args.bar_seconds)`
  - Save to `{output_dir}/processed/candles/{platform}/{market_id}_candles.parquet`
  - Directory structure: `data/processed/candles/kalshi/` and `data/processed/candles/polymarket/`
  - This matches the aligner's `candles_dir` parameter which expects `{candles_dir}/kalshi/{id}_candles.parquet`
- Log: "Reconstructed candles for {N} Kalshi markets, {M} Polymarket markets"

**Stage 4: Align and validate**
- Call `align_all_pairs(pairs, candles_dir=Path(output_dir) / "processed" / "candles", bar_seconds=args.bar_seconds)`
- Save aligned DataFrame to `data/processed/aligned_pairs.parquet` (snappy compression)
- Save quality report to `data/processed/data_quality_report.json`
- Log summary: "Aligned {N} pairs, excluded {M} pairs. Output: {rows} rows, {cols} columns"
- Log exclusion breakdown from quality report

**CLI arguments** (use argparse):
- `--pairs-file`: Path to accepted_pairs.json (default: `data/processed/accepted_pairs.json`)
- `--output-dir`: Base output directory (default: `data`)
- `--skip-fetch`: Skip Stage 2 (use existing raw trades on disk)
- `--bar-seconds`: Candle bar size in seconds (default: 14400 = 4 hours)

**Logging setup**: Configure root logger to INFO level with format `%(asctime)s %(levelname)s %(name)s: %(message)s`. Time each stage with `time.time()` and log duration.

**Error handling**: Wrap each market's reconstruction in try/except, log errors but continue. At the end, report total errors.
