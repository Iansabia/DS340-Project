Create `src/models/ppo_filtered.py` with class `PPOFilteredPredictor(BasePredictor)`.

Implementation per CONTEXT.md locked decisions:

**Design approach:** Extend SpreadTradingEnv (or wrap it) with reward masking. Create a `FilteredTradingEnv` subclass or wrapper that:
- Takes a boolean anomaly_flags array (per-bar, per-pair) from the autoencoder
- On step(): if current bar is NOT flagged, override reward to -0.01 (non_flagged_reward)
- If current bar IS flagged, use the normal reward (position * spread_change - tx_cost)
- This forces PPO to only earn/lose on anomalous timestamps

**Constructor:**
- Same PPO hyperparams as PPO-Raw: total_timesteps=100_000, policy_kwargs=dict(net_arch=[64, 64]), lr=3e-4, n_steps=256, batch_size=64, n_epochs=4, gamma=0.99, gae_lambda=0.95, clip_range=0.2, ent_coef=0.005, lookback=6, transaction_cost=0.02, random_state=42
- PLUS: anomaly_detector (AnomalyDetectorAutoencoder or None), non_flagged_reward=-0.01

**name property:** Returns "PPO-Filtered"

**fit(X_train, y_train):**
- Guard: raise ValueError if "group_id" not in X_train.columns
- set_seed(self._random_state)
- Determine feature_cols, bool_cols
- If anomaly_detector is None: create and train AnomalyDetectorAutoencoder on X_train
- Fit scaler via fit_feature_scaler()
- Compute anomaly flags for training data: self._anomaly_detector.flag_anomalies(X_train)
- Create FilteredTradingEnv (subclass/wrapper of SpreadTradingEnv) that applies reward masking using anomaly flags
- Create and train SB3 PPO (same config as PPO-Raw) on FilteredTradingEnv
- Cache everything for predict
- self._fitted = True, return self

**predict(X):**
- Same as PPORawPredictor.predict(): warm-up stitching, run policy deterministic, map actions to {-0.03, 0.0, +0.03}
- predict does NOT use the anomaly filter -- it runs the learned policy on all rows
- The filtering only affected training reward, not inference

**Implementation note:** Consider sharing code with PPORawPredictor where possible (helper functions for warm-up stitching, action mapping). But do NOT inherit from PPORawPredictor -- keep them independent for clarity.
