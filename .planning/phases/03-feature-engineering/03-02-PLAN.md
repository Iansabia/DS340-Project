From src/features/schemas.py (created in Plan 01):
```python
MIN_HOURS_THRESHOLD = 10

FEATURE_COLUMNS = [
    "timestamp",           # int: Unix seconds UTC (hourly-aligned)
    "pair_id",             # str: unique pair identifier
    "kalshi_close",        # float: Kalshi close price (trade or bid-ask midpoint)
    "polymarket_close",    # float: Polymarket close price
    "spread",              # float: kalshi_close - polymarket_close
    "kalshi_volume",       # float: Kalshi trade volume (0 if no trades)
    "polymarket_volume",   # float: Polymarket trade volume (0 if no trades)
    "bid_ask_spread",      # float: Kalshi yes_ask - yes_bid (NaN if unavailable)
    "kalshi_velocity",     # float: kalshi_close[t] - kalshi_close[t-1]
    "polymarket_velocity", # float: polymarket_close[t] - polymarket_close[t-1]
]
```

Feature matrix parquet schema:
- Sorted by (pair_id, timestamp)
- Each row = one hour for one matched pair
- pair_id is a string like "kxuejpn26feb2.1-0x4f0e5c99"
- timestamp is Unix seconds, hourly aligned (divisible by 3600)

PyTorch Forecasting TimeSeriesDataSet requirements:
- time_idx: integer column, monotonically increasing per group (0, 1, 2, ...)
- group_ids: list of column names identifying each series (use ["pair_id"])
- target: column name for the prediction target (use "spread")
- time_varying_known_reals: features known at prediction time
- time_varying_unknown_reals: features only known up to current time
