<objective>
Build the core feature engineering pipeline: load matched pairs, align hourly timestamps across platforms, compute microstructure features (spread, volume, bid-ask spread, price velocity), filter low-liquidity markets, and save a consolidated feature matrix to parquet.

Purpose: Transform raw per-platform OHLCV parquet files into a single time-aligned feature dataset ready for model consumption. This is the data backbone for all downstream model tiers.

Output: `data/processed/feature_matrix.parquet` containing hourly feature vectors for all valid matched pairs, plus `data/processed/build_report.json` documenting filtering decisions.
</objective>

    1. Create `src/features/alignment.py` with:
       - `align_pair_hourly(kalshi_df, polymarket_df, pair_id) -> pd.DataFrame`:
         a. Convert both DataFrames' 'timestamp' column to hourly floor: `ts // 3600 * 3600` (normalize to hour boundaries)
         b. For Kalshi: if 'close' column is all null but 'yes_bid_close' and 'yes_ask_close' exist, compute close as `(yes_bid_close + yes_ask_close) / 2` (bid-ask midpoint fallback)
         c. For each platform, keep the LAST row per hour (in case of duplicates from sub-hourly data)
         d. Inner-merge on 'timestamp' to get only hours present in BOTH platforms
         e. Forward-fill remaining null close prices within each platform (up to 6 hour gap max)
         f. Drop rows where either platform still has null close after forward-fill
         g. Add 'pair_id' column
         h. Return merged DataFrame with columns: timestamp, pair_id, kalshi_close, polymarket_close, kalshi_volume, polymarket_volume, kalshi_bid_close, kalshi_ask_close
         i. If no overlapping timestamps, return empty DataFrame with correct columns

    2. Create `src/features/engineering.py` with:
       - `compute_features(aligned_df) -> pd.DataFrame`:
         a. spread = kalshi_close - polymarket_close
         b. bid_ask_spread = kalshi_ask_close - kalshi_bid_close (NaN if either is null)
         c. kalshi_velocity = kalshi_close.diff() (NaN for first row per pair)
         d. polymarket_velocity = polymarket_close.diff() (NaN for first row per pair)
         e. Return DataFrame with columns matching FEATURE_COLUMNS from schemas.py
         f. The first row per pair_id will have NaN velocity (expected, downstream models handle this)

       - `filter_low_liquidity(pairs_data: list[tuple[str, pd.DataFrame]], min_hours=MIN_HOURS_THRESHOLD) -> tuple[list[tuple[str, pd.DataFrame]], list[dict]]`:
         a. For each (pair_id, aligned_df) tuple, count non-null close hours per platform
         b. If EITHER platform has fewer than `min_hours` non-null hours, exclude the pair
         c. Return (kept_pairs, filter_report) where filter_report is a list of dicts with pair_id, kalshi_hours, poly_hours, and reason for exclusion
         d. Log filtering stats

         b. For each valid pair, read both parquet files, call align_pair_hourly
         c. Call filter_low_liquidity on aligned results

         e. Concatenate all feature DataFrames

       - `main()` with argparse: `--pairs-path` (default data/processed/accepted_pairs.json), `--kalshi-dir` (default data/raw/kalshi), `--polymarket-dir` (default data/raw/polymarket), `--output-dir` (default data/processed)
