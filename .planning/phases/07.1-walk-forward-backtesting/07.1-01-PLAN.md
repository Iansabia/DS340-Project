Create `src/evaluation/backtester.py` with a `WalkForwardBacktester` class. Implementation spec from CONTEXT.md decisions:

**Constructor parameters:**
- `initial_capital: float = 10_000.0`
- `position_size: float = 100.0` (dollars per trade)
- `entry_cost_pp: float = 0.03` (3pp entry cost)
- `exit_cost_pp: float = 0.02` (2pp exit cost)
- `threshold: float = 0.02` (min abs prediction to trade)

**Core method `run(df: pd.DataFrame, predictions: np.ndarray) -> dict`:**

1. **Input:** df must contain `timestamp`, `pair_id`, `spread` columns. predictions is a 1-D array aligned row-for-row with df. df must already have spread_change_target computed (caller responsibility, same as run_baselines pattern).

2. **Compute spread_change_target if not present:** Use the same groupby/shift pattern as run_baselines: `df.groupby('pair_id')['spread'].shift(-1) - df['spread']`. Drop NaN target rows and corresponding predictions.

3. **Walk-forward loop:**
   - Sort df by timestamp (primary), pair_id (secondary)
   - Group rows by unique timestamp (each timestamp = one bar)
   - At each bar, for each pair: if `|prediction| > threshold`, enter trade
   - Position direction = sign(prediction)
   - Mid-price = (kalshi_close + polymarket_close) / 2
   - Number of contracts = position_size / mid_price (handle mid_price=0 by skipping)
   - Gross P&L = num_contracts * actual_spread_change
   - Entry cost = num_contracts * entry_cost_pp
   - Exit cost = num_contracts * exit_cost_pp
   - Net P&L = gross_pnl - entry_cost - exit_cost
   - Record: timestamp, pair_id, direction, net_pnl, gross_pnl, fees

4. **Aggregate to daily returns:**
   - Group trade net P&L by calendar day (timestamp // 86400)
   - Daily dollar return = sum of net P&L for that day
   - Daily percentage return = daily_dollar_return / portfolio_value_at_start_of_day
   - Portfolio value updates: start at initial_capital, accumulate daily dollar returns
   - Include zero-return days (days with no trades)

5. **Compute metrics:**
   - `annualized_sharpe`: mean(daily_pct_returns) / std(daily_pct_returns, ddof=1) * sqrt(365). Return 0.0 if std==0 or <2 days.
   - `total_pnl`: sum of all net P&L (dollars)
   - `max_drawdown`: peak-to-trough percentage decline of the equity curve
   - `calmar_ratio`: annualized_return / max_drawdown (0.0 if max_drawdown==0)
   - `win_rate`: fraction of trades with net_pnl > 0
   - `num_trades`: total trades taken
   - `avg_trade_duration_hours`: always 4.0 (single-bar hold)
   - `total_fees`: sum of all entry + exit costs
   - `equity_curve`: list of (day_ordinal, portfolio_value) for plotting
   - `daily_returns`: list of daily percentage returns (for external analysis)

6. **Return dict** with all metrics above.

**Also add a convenience function:**
```python
def compute_break_even_fee(df, predictions, initial_capital=10_000, position_size=100, threshold=0.02) -> float:
```
Binary search for the round-trip fee level (entry + exit) where Sharpe drops to ~0. Start from 0pp to 20pp. Return the break-even fee in percentage points.
