# Phase 7.1: Walk-Forward Backtesting — Context

**Why this phase exists:** Current Sharpe ratios (~15-23 annualized) are inflated because (a) returns are in absolute spread units with no capital normalization, (b) panel data cross-pair observations treated as sequential bars, and (c) no transaction costs. A walk-forward portfolio simulator produces the honest, paper-ready Sharpe.

<domain>
## Phase Boundary

**In scope:**
- Walk-forward portfolio simulator that steps through test data chronologically
- Realistic transaction costs (Kalshi ~5pp, Polymarket ~2pp)
- Capital-normalized dollar returns → proper annualized Sharpe
- Equity curve, max drawdown, Calmar ratio
- Run all 8 models through the simulator
- Update the cross-tier comparison table with backtested metrics

**Out of scope:**
- K-fold time-series cross-validation (stretch goal, not blocking)
- Retraining models (use existing trained models)
- Modifying existing profit_sim.py (keep both — simulated and backtested)
</domain>

<decisions>
## Implementation Decisions (LOCKED)

### Walk-Forward Simulator
- **Steps through test bars in timestamp order** (not pair order)
- At each 4-hour timestamp: evaluate all active pairs, decide trades
- **Initial capital:** $10,000
- **Position sizing:** Equal-dollar per trade — invest $100 per contract at mid-price (kalshi_close + polymarket_close) / 2
- **Max simultaneous positions:** No limit (all active pairs can be traded)
- **Transaction costs per trade:**
  - Entry: 3pp of contract price (covers Kalshi bid-ask + Polymarket gas)
  - Exit: 2pp of contract price (less slippage on exit)
  - Total round-trip: ~5pp per position
- **Position management:** Enter at start of bar based on model prediction, exit at end of bar (hold for 1 bar = 4 hours). No multi-bar positions for simplicity.
- **Returns:** Dollar P&L per bar / portfolio value at start of bar = percentage return
- **Sharpe:** Computed on daily percentage returns, annualized by sqrt(365)

### Metrics to produce
- **Annualized Sharpe** (on daily dollar returns — THE honest number)
- **Total dollar P&L** (after costs)
- **Max drawdown** (peak-to-trough % decline)
- **Calmar ratio** (annualized return / max drawdown)
- **Win rate** (% of trades with positive P&L after costs)
- **Number of trades**
- **Average trade duration** (always 1 bar = 4h in this version)
- **Break-even fee level** (at what cost does Sharpe drop to 0?)

### File structure
- `src/evaluation/backtester.py` — WalkForwardBacktester class
- `experiments/run_backtest.py` — script that runs all 8 models through backtester
- `experiments/results/backtest/` — per-model backtest JSONs
- `experiments/figures/backtest_equity_curves.png` — equity curves for all models
- `experiments/figures/backtest_drawdown.png` — drawdown chart

### Integration
- Models already trained in Phases 5-6 — just need to call predict()
- Reuse `prepare_xy` / `prepare_xy_for_seq` from run_baselines.py for data loading
- Save results in same JSON schema as tier1/2/3 but with additional backtest-specific fields (max_drawdown, calmar_ratio, dollar_pnl)
- Add `--backtest` flag to run_baselines.py OR keep as separate script

### What the paper reports
- Replace the inflated Sharpe column in the comparison table with backtested Sharpe
- Keep per-trade Sharpe as a model-quality metric alongside
- Equity curve figure goes in the paper as the main trading result visual
</decisions>
