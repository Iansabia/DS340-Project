## Decisions Made
- **Kalshi orderbook API over pmxt**: pmxt.Kalshi() returns price=0 for all markets. Direct orderbook API at `/markets/{ticker}/orderbook` returns real bid/ask data for active markets.
- **Polymarket Gamma API over pmxt paginated**: Our pair IDs are hex clobTokenIds (e.g., `0x89655f46...`). pmxt uses numeric market_ids (e.g., `1640919`). Converting hex to decimal and querying Gamma API directly is the reliable mapping path.
- **Snapshot bar mode**: Since we poll once per 4h interval, OHLCV fields all equal the single price snapshot. Volume/trade fields are zero. This is honest representation of the data granularity.
- **Demo mode**: All 144 pairs are historical and resolved, so live APIs return no prices. `--demo` flag creates synthetic bars with fabricated prices for pipeline verification.
