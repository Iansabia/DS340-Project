Create `src/live/collector.py` with class `LiveCollector`:

**Constructor:**
- Loads pair mapping from `data/processed/all_pairs_v2.json` (all 3079 pairs, but only those whose pair_id appears in train.parquet's 144 unique pair_ids are active)
- Builds a lookup: `{pair_id: {kalshi_market_id, polymarket_market_id}}` for the 144 active pairs
- Accepts `live_dir: Path = Path("data/live")` for output

**Price fetching methods:**

`fetch_polymarket_prices() -> dict[str, float]`:
- Use `pmxt.Polymarket().fetch_markets_paginated()` to page through ALL markets
- Build a dict: `{market_id: yes_price}` for markets where `yes` is not None and `0 < price < 1`
- Handle pagination via `next_cursor` until all markets fetched (or a reasonable limit like 10000)
- Handle `pmxt.RateLimitExceeded` with 10-second backoff retry (up to 3 retries)
- Handle `pmxt.PmxtError` and `pmxt.NetworkError` by logging and returning partial results

`fetch_kalshi_prices() -> dict[str, float]`:
- For each of the 144 active pairs' kalshi_market_id, call:
  `requests.get(f"https://api.elections.kalshi.com/trade-api/v2/markets/{ticker}/orderbook", params={"depth": 1})`
- Parse the orderbook: if `no_dollars` has entries, best yes price = `1.0 - float(no_dollars[0][0])`
  If `yes_dollars` has entries, best yes price = `float(yes_dollars[0][0])`
  If both sides have entries, use midpoint: `(best_yes_bid + (1.0 - best_no_ask)) / 2`
  If neither side has entries, return None for this market
- Rate limit: sleep 0.06s between requests (16.7 req/s, below Kalshi's 20 req/s limit)
- Handle 404 (market not found / resolved) by skipping
- Handle other HTTP errors by logging and skipping
- Return `{kalshi_market_id: price}`

IMPORTANT: The Polymarket `market_id` from pmxt is a numeric string (e.g., "1640919"), but our all_pairs_v2.json has Polymarket IDs as hex token IDs (e.g., "0x4f0e5c99..."). The collector must handle this mapping. Strategy:
- First, try to use `pmxt.Polymarket().fetch_market({'condition_id': poly_market_id})` per pair
- If that doesn't work (pmxt may not support condition_id lookups), fall back to:
  Use the Polymarket Gamma API directly: `GET https://gamma-api.polymarket.com/markets?clob_token_ids={token_id}` returns market data with `outcomePrices`
- Document whichever approach works in pair_mapping.json

**Bar construction:**

`build_snapshot_bar(kalshi_price: float, polymarket_price: float, pair_id: str, timestamp: int) -> dict`:
- For first version (snapshot = close price), set:
  - `kalshi_vwap = kalshi_open = kalshi_high = kalshi_low = kalshi_close = kalshi_price`
  - Same pattern for polymarket
  - `kalshi_volume = polymarket_volume = 0.0` (no volume data from snapshot)
  - `kalshi_trade_count = polymarket_trade_count = 0.0`
  - `kalshi_dollar_volume = polymarket_dollar_volume = 0.0`
  - `kalshi_buy_volume = kalshi_sell_volume = 0.0`
  - `polymarket_buy_volume = polymarket_sell_volume = 0.0`
  - `kalshi_realized_spread = polymarket_realized_spread = 0.0`
  - `kalshi_max_trade_size = polymarket_max_trade_size = 0.0`
  - `kalshi_has_trade = polymarket_has_trade = False`
  - `kalshi_hours_since_last_trade = polymarket_hours_since_last_trade = 0.0`
  - `spread = kalshi_price - polymarket_price`
  - `pair_id = pair_id`
  - `timestamp = timestamp`

`assemble_bar_dataframe(bars: list[dict]) -> pd.DataFrame`:
- Convert bars to DataFrame with ALIGNED_COLUMNS
- Call `compute_derived_features()` from `src.features.engineering` to add the 6 derived columns (37 total)
- Add `time_idx`: load existing bars.parquet, find max time_idx, increment. If no existing file, start at 0.
- Add `group_id`: map pair_id to integer. Load existing mapping or build from train.parquet's group_id mapping.
- Ensure dtypes match train.parquet: float64 for numeric, bool for has_trade, int64 for timestamp/time_idx/group_id, object for pair_id
- Return 39-column DataFrame

`append_to_parquet(df: pd.DataFrame, path: Path)`:
- If path exists, read existing, pd.concat, write back
- If not, write new
- Always sort by (pair_id, time_idx) before writing

**Pair mapping:**

`build_pair_mapping() -> dict`:
- For each of the 144 active pairs, record:
  `{pair_id: {kalshi_market_id, polymarket_market_id, kalshi_ticker_for_api, polymarket_lookup_id}}`
- Save to `data/live/pair_mapping.json`

`collect_once() -> int`:
- Main collection cycle. Returns number of bars collected.
- Fetch Kalshi prices and Polymarket prices
- For each active pair, look up both prices. Skip if either is None.
- Build snapshot bars for all pairs with both prices
- Assemble into DataFrame, append to bars.parquet
- Log: "Collected {n} bars for {n} pairs at {timestamp}"
