# Phase 7.2: Live Paper Trading and Data Collection

**Purpose:** Deploy models on live prediction market data. Paper trade, collect data, retrain. Show real-world viability.

<domain>
## Phase Boundary

**In scope:**
- Live data collector polling Kalshi + Polymarket every 4 hours via pmxt SDK
- 4h bar construction from live prices (OHLCV + microstructure features)
- Growing dataset stored in data/live/
- Paper trading engine running all models on live data
- Trade logging with timestamps, predictions, actuals, paper P&L
- Weekly auto-retrain on original + live data
- Performance tracking dashboard (CLI-based)

**Out of scope:**
- Real money execution (paper only for now)
- New model architectures
- Web UI (CLI is sufficient)
- Alerting (Slack/email) — add later if paper trading proves profitable
</domain>

<decisions>
## Implementation Decisions (LOCKED)

### Data collector
- **Poll interval:** Every 4 hours (matching our bar size)
- **Source:** pmxt SDK — `pmxt.Kalshi()` and `pmxt.Polymarket()` for unified market data
- **Pairs:** All 144 currently matched pairs (from data/processed/all_pairs_v2.json, filtered to those with aligned data)
- **Bar construction:** At each poll, fetch current price → store. Compute OHLCV from 4h window of polls (or use single snapshot as close price). For first version, snapshot = close price, volume from API if available.
- **Feature engineering:** Reuse `src/features/engineering.py` to compute derived features (spread, price_velocity, volume_ratio, spread_momentum, spread_volatility, order_flow_imbalance)
- **Storage:** Append to `data/live/bars.parquet` (same schema as data/processed/train.parquet)
- **Pair ID mapping:** `data/live/pair_mapping.json` maps pmxt market_id → our pair_id

### Paper trading engine
- **Models:** All 8 (Naive, Volume, LR, XGBoost, GRU, LSTM, PPO-Raw, PPO-Filtered)
- **Initial weights:** Trained on existing data/processed/train.parquet
- **Inference:** At each 4h bar, run all models → generate predictions → log paper trades
- **Trade decision:** Same as evaluate() — trade when |prediction| > threshold (0.02)
- **Paper P&L:** Track per-model cumulative P&L, win rate, trade count
- **Trade log:** `data/live/paper_trades.jsonl` (one JSON line per trade event)
- **No actual execution** — log only

### Auto-retrain
- **Trigger:** Manual for now (run a script). Can be cron'd later.
- **Data:** Combine data/processed/train.parquet + data/live/bars.parquet
- **Retrain all 8 models** on combined dataset
- **Save new weights** to `models/live/` directory
- **Compare:** Did more data help? Log retrain metrics to `data/live/retrain_log.json`

### Scheduling
- **macOS launchd** or a simple `while true; do ... sleep 14400; done` loop
- For development: manual trigger via `python -m src.live.collector`
- For production: cron entry `0 */4 * * * cd ~/Desktop/DS340\ Project && .venv/bin/python -m src.live.collector`

### File structure
```
src/live/
  __init__.py
  collector.py       — polls pmxt, constructs bars, appends to parquet
  paper_trader.py    — loads models, runs inference, logs trades
  retrain.py         — combines data, retrains all models
  dashboard.py       — prints current paper P&L summary

data/live/
  bars.parquet       — growing dataset of live 4h bars
  paper_trades.jsonl — trade log (one line per trade event per model)
  pair_mapping.json  — pmxt market_id → our pair_id
  retrain_log.json   — history of retrain events + metric deltas

models/live/         — retrained model weights
```

### Key constraints
- pmxt SDK is already installed
- No new deps needed (pandas, torch, sklearn all present)
- Must not modify existing src/models/ or src/evaluation/ — live code is additive
- Must handle API rate limits gracefully (pmxt has per-10s limits)
- Must handle pairs where one or both platforms have no current price (skip gracefully)
</decisions>
